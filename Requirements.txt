Absolutely. Here are two easy ways to get it running in VSCode.

# Option A (recommended): Next.js + Tailwind + shadcn/ui

This matches the prototype’s imports (`@/components/ui/...`) out of the box.

1. **Scaffold the app**

```bash
npx create-next-app@latest daily-divination --typescript --tailwind --eslint
cd daily-divination
```

2. **Initialize shadcn/ui**

```bash
npx shadcn@latest init -d
```

3. **Add the components the prototype uses**

```bash
npx shadcn@latest add button card input label sheet select textarea badge
```

4. **Install extras**

```bash
npm i framer-motion lucide-react
```

5. **Open in VSCode**

```bash
code .
```

6. **Drop in the code**

* Open `app/page.tsx` (or create a new route if you prefer).
* Paste the prototype code I put in the canvas.
* It uses the `@/components/ui/*` paths that shadcn just generated.

7. **Run it**

```bash
npm run dev
```

Then visit `http://localhost:3000`.

# Option B: Vite + React + Tailwind (works fine too)

If you prefer Vite, just add a small alias so `@/...` resolves to `src/...`.

1. **Scaffold**

```bash
npm create vite@latest daily-divination -- --template react-ts
cd daily-divination
```

2. **Tailwind setup**

```bash
npm i -D tailwindcss postcss autoprefixer
npx tailwindcss init -p
```

* In `tailwind.config.js` set:

```js
content: ["./index.html", "./src/**/*.{ts,tsx}"]
```

* In `src/index.css` add:

```css
@tailwind base;
@tailwind components;
@tailwind utilities;
```

3. **shadcn/ui + components**

```bash
npm i class-variance-authority tailwind-merge tailwindcss-animate lucide-react framer-motion
npx shadcn@latest init -d
npx shadcn@latest add button card input label sheet select textarea badge
```

4. **Add an alias for `@/`**

* In `tsconfig.json`:

```json
{
  "compilerOptions": {
    "baseUrl": ".",
    "paths": { "@/*": ["src/*"] }
  }
}
```

* (Optional) In `vite.config.ts`:

```ts
import path from "path";
resolve: { alias: { "@": path.resolve(__dirname, "src") } }
```

5. **Create the app file**

* Replace `src/App.tsx` with the canvas code.
* Ensure the imports like `@/components/ui/button` exist (they will after `shadcn add`).

6. **Run**

```bash
npm run dev
```

Visit `http://localhost:5173`.

## Quick troubleshooting

* **UI imports not found**: Re-run the `shadcn add ...` commands (Next.js: they land in `components/ui`; Vite: in `src/components/ui`).
* **Tailwind classes don’t render**: confirm Tailwind content globs cover your files and you added `@tailwind base; @tailwind components; @tailwind utilities;`.
* **Path alias `@/` not resolving (Vite)**: verify `tsconfig.json` paths and (if needed) the Vite alias in `vite.config.ts`.
* **Node version**: use Node 18+.
